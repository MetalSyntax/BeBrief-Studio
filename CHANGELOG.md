## [2.1.0] - 2026-07-01

### Added
- **5 new section types** ‚Äî `testimonial`, `team`, `awards`, `tech-stack` and `timeline`, each with a canvas component, dedicated inspector editor, HTML export case + CSS, and i18n in ES/EN/PT.
- **Section manifest (`src/lib/sections/manifest.ts`)** ‚Äî Single source of truth for all 15 section types' icon, label and default data, consumed by `SectionList`'s add-menu and `projectStore.addSection` instead of duplicated switches.
- **Schema versioning for `.bbs.json`** ‚Äî `jsonPortability.ts` now walks a migration chain keyed by `_schema` version and surfaces a warning (via optional `onWarning` callback) instead of silently importing an unrecognized/future schema.
- **CI pipeline** ‚Äî `.github/workflows/ci.yml` runs lint, typecheck + build, Jest, and a Cypress e2e job against a production preview build on every push/PR to `main`.
- **Privacy Policy & Terms of Service** ‚Äî `LegalModal` component with full ES/EN/PT content reflecting the app's actual local-first behavior (no accounts, no backend, third-party requests disclosed). Linked from the Dashboard footer.
- **First-run onboarding** ‚Äî 4-step dismissible `OnboardingModal`, shown once per browser (`localStorage` flag) with a replay button in the Dashboard header.
- **Working share/preview links** ‚Äî `?share=` URLs (generated by the existing `ShareModal`) now actually render a read-only preview (`SharedPreviewView`, sandboxed iframe of the exported HTML) with a "Duplicate & edit" CTA, instead of silently loading the normal dashboard.
- **Dashboard visual redesign** ‚Äî Hero section with decorative glow and live stat chips (section types / themes / saved projects), pill-style segmented tabs, per-project theme-accent color dot, and a restructured footer (logo + tagline, Legal / Creator columns, dynamic copyright bar).
- **Community gallery templates fleshed out** ‚Äî The 3 bundled community templates now have 6 sections each (was 2‚Äì3) for a more representative preview when duplicated.

### Fixed
- **XSS in HTML exporter** ‚Äî `wrap()` and several raw interpolations (image `src`, link `href`, inline `style="font-family"`, footer text) never escaped user-provided content. Added `escapeHtml()` / `sanitizeUrl()` helpers and applied them throughout `htmlExporter.ts`.
- **Community templates unreadable in light themes** ‚Äî All 3 bundled templates set `textColor: 'var(--text-main)'`, a CSS custom property that doesn't exist in the theme token set (the real one is `--text`), leaving text color undefined/inherited. Most noticeable in the Nordic Forest (light theme) template.
- **PWA update banner hardcoded strings** ‚Äî Moved to `pwa.*` i18n keys (ES/EN/PT) instead of hardcoded Spanish.
- **Dashboard unclosed `<section>` tag** ‚Äî Was breaking `tsc`/`vite build`.
- **Missing `compressAndEncodeProject` import in `Dashboard.tsx`** ‚Äî The community gallery's "Previsualizar" button threw a `ReferenceError` on click.
- **Invalid `theme: 'nordic'` in community templates** ‚Äî Not a valid `ThemeId`; corrected to `'nordic-light'`.
- Assorted pre-existing `tsc` failures picked up while touching these files: unused imports in `CommentsPanel.tsx`, a type-only import violation in `Dashboard.tsx`.

### Changed
- **`package.json` / `src-tauri/tauri.conf.json` / `src-tauri/Cargo.toml` version** ‚Äî Bumped to `2.1.0` (previously out of sync at `2.0.0` / `1.0.0` / `0.1.0`).

## [2.0.0] - 2026-06-22

### Added
- **JSON Portability** ‚Äî Export any project to a versioned `.bbs.json` file (`exportProjectToJSON`, `downloadProjectJSON`) and import it back on any device (`parseProjectFromJSON`). Schema version `bbs-v1` with validation.
- **Dashboard Import / Export buttons** ‚Äî Global "Importar .bbs.json" button in the dashboard header with hidden file input. Per-project "Export JSON" button revealed on card hover (Download icon).
- **`importProject` store action** ‚Äî Adds the imported project to the projects array, remints the ID on collision, saves to LocalStorage and navigates to the editor.
- **Dedicated inspector editors for all 10 section types** ‚Äî Problem, Process, Typography, Results and UX Flow now have full content editors in `SectionInspector` instead of the generic fallback.
- **Process step icon picker** ‚Äî 42-icon grid (5 columns, scrollable) using `CustomSelect` for each step's icon field in the Process inspector.
- **Typography font family picker** ‚Äî Restricted to the 8 permitted Google Fonts (Montserrat, Sora, Manrope, Space Grotesk, DM Sans, Inter, Bebas Neue, Playfair Display).
- **Per-section display font override** ‚Äî `displayFont` field in `SectionStyle`; wired in the inspector's CustomSelect and applied as a CSS variable `--font-display` inline override both in `SectionRenderer` and in the HTML exporter.
- **Duplicate section** ‚Äî Copy icon button in the sidebar row; `duplicateSection(sectionId)` store action deep-clones the section, inserts it after the original and sets it as active.
- **Toast notification system** ‚Äî `ToastProvider` context with `useToast()` hook returning `{ success, error, info }`. Auto-dismiss at 4 s, fixed bottom-right stack. All `alert()` calls replaced.
- **Keyboard shortcuts** ‚Äî `useKeyboardShortcuts` hook: `Cmd/Ctrl+Z` undo, `Cmd/Ctrl+Shift+Z` redo, `Escape` deselects the active section.
- **`THEME_OPTIONS` single source of truth** ‚Äî Exported from `src/lib/themes/index.ts`; Toolbar and SectionInspector both build their theme dropdowns from it instead of duplicating the list.
- **Mobile tab bar i18n** ‚Äî Labels for Sections / Canvas / Inspector tabs go through `react-i18next`.
- **PWA Google Fonts caching** ‚Äî Two Workbox runtime cache rules added to `vite.config.ts`: `StaleWhileRevalidate` for `fonts.googleapis.com` stylesheets, `CacheFirst` for `fonts.gstatic.com` font files (1-year TTL, 30 entries).
- **`<html lang>` dynamic attribute in exporter** ‚Äî `exportProjectToHTML(project, lang)` now writes the detected locale into `<html lang="...">` instead of the hardcoded `"es"`.

### Fixed
- **Image export null element** ‚Äî `getElementById('brief-render-canvas')` silently returned `null`. Corrected to `'brief-canvas-export'` (actual element ID in `EditorCanvas`).
- **Tauri binary image corruption** ‚Äî `writeTextFile` re-encoded raw bytes as text, corrupting PNG/WebP. Fixed by using `writeFile(filePath, bytes)` with the raw `Uint8Array`.
- **Broken template literal in HTML exporter** ‚Äî A stray backtick after `<html lang="${lang}">` closed the template string prematurely, causing ~300 TypeScript parse errors across the entire file.
- **EditableText double-trigger** ‚Äî Normalized newlines (`\r\n` ‚Üí `\n`) and trimmed trailing whitespace in `handleBlur` and the sync `useEffect` before comparing values, preventing spurious `onChange` calls.
- **Auto-zoom desktop overflow** ‚Äî Desktop mode previously set `width: '100%'` and `transform: none`, allowing 1600px sections to overflow narrower viewports. Now always applies `Math.min(1, parentWidth / 1600)` scale-to-fit.
- **OCR worker init (Tesseract.js v7)** ‚Äî Removed deprecated `loadLanguage` / `initialize` shims; switched to the v7 `createWorker('spa+eng+por', 1, { logger })` three-argument API.
- **`deleteProject` alert in store** ‚Äî Store cannot access the toast context. Moved the guard (projects ‚â§ 1) to `Dashboard.handleDeleteProject` which shows a toast; the store action returns early silently.
- **Hardcoded colors in HTML exporter** ‚Äî Five instances of `color: #fff` replaced with `color: var(--text)` (metric-value, step-title, font-name, color-name, footer-author) so light themes render correctly.
- **`object-contain` typo** ‚Äî `object-contain: contain` corrected to `object-fit: contain` in the mockups exporter CSS.
- **Portuguese i18n typo** ‚Äî `"Tamanhoo"` ‚Üí `"Tamanho"` in `pt.json` `inspector.field.titleSize`.
- **React import missing in `SectionRenderer`** ‚Äî Added `import React from 'react'` after using `React.CSSProperties` and `React.ReactNode`.

### Changed
- **`package.json` version** ‚Äî Bumped from `0.0.0` to `2.0.0`.
- **`EditorCanvas` auto-zoom** ‚Äî Simplified wrapper div to always use scaled pixel dimensions; removed the conditional `width: 100%` branch.
- **Process section icon picker** ‚Äî Expanded from 9 to 42 icons in a 5-column scrollable grid (14px icons, `w-52 max-h-48 overflow-y-auto`).
- **Cypress test** ‚Äî Replaced `cy.url().should('include', '/editor')` with `cy.get('#brief-canvas-export').should('exist')` since the SPA has no router.
- **V2_PLAN.md moved** ‚Äî Roadmap document relocated from the project root to `.plans/V2_PLAN.md`.
